# Как посмотреть результаты

## Текущий статус

Сейчас запущен **фронтенд-интерфейс** с демонстрационными данными. Для получения реальных данных нужно настроить бэкенд с парсерами.

## Просмотр текущих результатов

### 1. Откройте приложение в браузере

После запуска `npm run dev` приложение доступно по адресу:
- **http://localhost:5173** (или другой порт, указанный в консоли)

### 2. Просмотр данных в интерфейсе

#### Главная страница (Dashboard)
- **Статистика**: общее количество тендеров, профилей, совпадений
- **Статус парсеров**: активность парсеров (Riigihangete, E-procurement, Hanked.ee)
- **Последние совпадения**: недавно найденные тендеры

#### Страница "Тендеры"
- Список всех тендеров с фильтрацией
- Примеры тендеров с hanked.ee по бетонным работам:
  - Бетонные работы для жилого комплекса (€125,000)
  - Заливка бетонного фундамента (€85,000)
  - Бетонные работы по реконструкции дороги (€250,000)

#### Страница "Профили клиентов"
- Профиль "Бетонные работы" уже настроен
- Категории: Строительство, Бетонные работы
- Регионы: Таллинн, Тарту, Нарва, Пярну
- Ключевые слова: бетон, бетонные работы, фундамент, заливка бетона

#### Страница "Архитектура"
- Визуализация архитектуры системы
- Показывает все источники данных, включая hanked.ee

## Для получения реальных данных

### ✅ Парсер для hanked.ee создан!

Парсер для hanked.ee готов к использованию. См. `backend/README.md` для инструкций.

### Быстрый запуск парсера:

```bash
# 1. Установите зависимости
cd backend
pip install -r requirements.txt

# 2. Запустите парсер
python parsers/hanked_parser.py
```

Результаты будут сохранены в файл `hanked_tenders.json`.

### Автоматический запуск:

```bash
# Запуск планировщика (парсинг каждые 30 минут)
python scheduler.py
```

### Необходимо настроить:

1. **✅ Парсер для hanked.ee** - ГОТОВ
2. ⏳ Парсер для Riigihangete Portal
3. ⏳ Парсер для E-procurement Portal

2. **База данных**
   - PostgreSQL для хранения тендеров
   - Redis для кэширования

3. **Очередь задач**
   - RabbitMQ или Redis для очереди сообщений

4. **Планировщик**
   - Cron или Celery для автоматического запуска парсеров

5. **API для фронтенда**
   - REST API для получения данных из базы
